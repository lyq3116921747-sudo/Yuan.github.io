我们可能有在超算上vibe coding 的需求，由于超算的网络无法连接外网，`module load proxy` 之后，代理服务器也只能访问内网，github连接也不稳定。这里记录了一下解决方法。（以上方法理论上可以推广到其他不能连接外网的服务器）
## ssh命令
申请完卡之后，我们会得到一个地址和端口：`701056.proxy.nscc-gz.cn:12345`（只是举个例子）
**我们使用下面的指令来进行ssh连接：**
```
ssh -p 12345 -R 1080:127.0.0.1:7897 sysu_zhangy_1@701056.proxy.nscc-gz.cn
```
这条命令做了这些事：
**通过 SSH 连接到远程服务器 `701056.proxy.nscc-gz.cn`（端口 `12345`），并建立一个 远程端口转发（`-R`），使得远程服务器上对 1080 端口的访问，都会被 通过 SSH 隧道转发回你本机的 `127.0.0.1:7897` 端口。**

简单点说，远程主机的网络请求体经过了下面的路径：
**远程服务器 → 自己1080端口 → 你的本地7897端口。**
